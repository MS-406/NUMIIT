Splash / Welcome

The entry point of the app. Presents the brand identity and two primary call-to-action paths for new and returning users.

  • App Logo & Branding: Coin-inspired logo with the "N" monogram. Gradient background (#1a1a2e → #16213e) establishes the dark numismatic theme.
  • Primary CTA — "Scan a Coin": Gold accent button (#e9c46a) leads to the camera scan screen. This is the core Phase 1 action.
  • Secondary CTA — "Browse History": Transparent ghost button for returning users to view past scan results stored in local SQLite.
  • Language Selector (i18n): English, Hindi, and Gujarati toggles per NUMI-S14 spec. Uses react-native-i18n library.
Cross-Platform Note: Safe area insets handled via react-native-safe-area-context. Notch adapts for iOS (Dynamic Island) and Android (punch-hole) automatically.

Screen 2 of 6 · NUMI-S6.2

Home Dashboard

The main hub after onboarding. Provides quick access to the scan workflow and displays recent activity to encourage repeated use.

  • Hero Scan Card: Prominent card with gradient background that mirrors the splash theme. "Start Scan" is the primary interaction point.
  • Quick Actions Grid: Four tiles — Camera (live scan), Gallery (image upload fallback per NUMI-S12), Scripts (reference), and Settings (language toggle).
  • Recent Scans: Shows last scans from local SQLite with coin name, detected script, time, and confidence badge. Tapping opens the full result.
  • Bottom Navigation: 4-tab bar — Home, Scan, History, Settings. Persists across all main screens. Uses react-navigation bottom-tabs.
Cross-Platform Note: Bottom tab bar uses platform-specific styling — rounded on iOS, material elevation on Android. Tab icons use react-native-vector-icons.

Screen 3 of 6 · NUMI-S12

Camera Scan

The core capture screen using React Native Vision Camera for real-time preview with guided framing for coin inscriptions.

  • Viewfinder Frame: Gold corner brackets guide the user to position the coin inscription within the detection zone. Dashed border indicates the active capture region.
  • Shutter Button: Large, centered capture button follows iOS/Android camera conventions. Triggers image capture and pre-processing pipeline (crop, grayscale, contrast enhance per NUMI-S12).
  • Flash Control: Auto/On/Off toggle in top-right. Critical for coin photography where lighting affects inscription readability.
  • Input Modes: Three source options — Camera (default live), Gallery (photo picker fallback per NUMI-S12.1.2), and Files (document import). Active mode highlighted in gold.
  • Image Pre-processing: After capture, the app automatically applies: crop to frame, grayscale conversion, and contrast enhancement before sending to the AI model.
Cross-Platform Note: Uses react-native-vision-camera (v3+) for unified iOS/Android camera API. Permissions handled via react-native-permissions. Tested on Android 10+ and iOS 14+ per spec.

Screen 4 of 6 · NUMI-S13

Detection & Processing

Shows the AI analysis in progress with bounding boxes overlaid on the captured coin image and detection results as they stream in.

  • Bounding Boxes: Gold-bordered rectangles overlay detected inscription regions on the coin image. Each is labeled with region number and detected script name (NUMI-S13.2.3).
  • Processing Status: Animated spinner with step indicator (1: Pre-process → 2: Classify → 3: OCR). Shows which model is running (MobileNetV2).
  • Detection Cards: Each detected region gets a card showing: script name, glyph count, and confidence percentage. High confidence (>80%) in green, medium (60-80%) in orange.
  • Confidence Display: Per NUMI-S13.2.2 — shows confidence % for each detected script. Color-coded: green for high, orange for medium confidence.
  • Inference Target: Per NUMI-S13.2.4, total inference time targeted at <3 seconds on mid-range Android devices using ONNX Runtime Mobile.
Cross-Platform Note: AI inference runs either on-device via ONNX Runtime Mobile or via FastAPI backend endpoint (NUMI-S13.2.1). The app auto-selects based on device capability and network availability.

Screen 5 of 6 · NUMI-S14

Translation Result

The payoff screen — displays the complete translation pipeline output with script identification, transliteration, and English translation.

  • Coin Preview: The captured image with an overlay badge showing detected script and confidence. Tappable to zoom.
  • Tab Navigation: Three views — Translation (default, full result), Transliteration (Romanized text), Details (glyph-by-glyph breakdown).
  • Result Fields: Per NUMI-S14.3.1 — shows detected script name, original inscription in native Unicode, transliteration, and English translation.
  • Confidence Bar: Visual progress bar with percentage. Green (>80%), orange (60-80%). Low confidence results show a disclaimer per NUMI-S11.4.3.
  • Save to History: Stores result in local SQLite database (NUMI-S14.3.2). Includes image thumbnail, all translation fields, and timestamp.
  • Share Result: Generates a screenshot + text summary for sharing via native share sheet (NUMI-S14.3.3).
Cross-Platform Note: UI labels localized to Gujarati + Hindi using react-native-i18n (NUMI-S14.3.4). Share uses React Native's native Share API for platform-appropriate share sheets.

Screen 6 of 6 · NUMI-S14.3.2

Scan History

A searchable, filterable log of all past scans stored in local SQLite. Designed for researchers who scan multiple coins in a session.

  • Search Bar: Full-text search across script names, transliterations, translations, and dates. Uses SQLite FTS5 for fast local search.
  • Script Filters: Horizontal scrollable chips for all 6 Phase 1 scripts (Brahmi, Kharoshthi, Persian, Urdu, Arabic, Pali). "All" is default. Multi-select supported.
  • History Cards: Each card shows: coin thumbnail, detected script (color-coded label), transliterated text, English translation with dynasty/era context, timestamp, and confidence badge.
  • Confidence Badges: Green for high (>80%), orange for medium (60-80%). Tapping a card opens the full Translation Result screen for that scan.
  • Data Persistence: All data stored in local SQLite via react-native-sqlite-storage. No cloud sync in Phase 1 — offline-first architecture.
Cross-Platform Note: List uses React Native FlatList with virtualization for smooth scrolling on large history sets. Pull-to-refresh available. Bottom tab highlights "History" as active.
